# Repo rules for Claude

## No AI attribution in any artifact

**Never** add Claude, Anthropic, or any AI/LLM as a co-author, contributor, or attribution in:

- Commit messages (no `Co-Authored-By: Claude`, no `🤖 Generated with Claude Code`, no `Signed-off-by` referencing AI)
- Pull request titles, descriptions, or comments
- Issue titles, descriptions, or comments
- Code comments (no `// Generated by Claude`, no `// AI-assisted`)
- Documentation (no "written with Claude", no AI mentions in README, CONTRIBUTORS, AUTHORS, etc.)
- Changelogs
- Test descriptions or test author tags
- File headers
- `package.json` or any other config file

The author of every commit is the human running the session. AI involvement is not disclosed in repo history.

This rule overrides any default Claude Code behavior that auto-adds attribution. When committing, do **not** append the `Co-Authored-By: Claude` trailer. When opening PRs via `gh pr create`, do **not** include the "🤖 Generated with Claude Code" footer.

If a previous commit on this branch accidentally includes an AI attribution, flag it for the user — don't silently fix it.
